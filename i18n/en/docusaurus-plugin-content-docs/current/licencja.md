- **The authors bear no responsibility** for data loss, security breaches, or system malfunctions
- **The user is fully responsible** for configuring, securing, and maintaining their instance
- **The authors have no access** to data stored on self-hosted instances

OpenKSeF operates on tax data (invoices from KSeF). Before production deployment, we recommend:

1. Security configuration review by a qualified person
2. Implementing HTTPS, backups, and monitoring
3. Compliance with data protection regulations (GDPR)
4. Regular updates to the latest versions

:::warning
OpenKSeF is in **beta**. Use in a production environment with real tax data is at the user's own risk and responsibility. There may be changes in the API and data structure between versions.
:::
